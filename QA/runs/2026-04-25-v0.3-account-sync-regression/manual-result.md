# Simulated Human QA Result

日期：2026-04-25

结论：通过，带环境限制。

本轮按“真实用户视角”复核 v0.3 账号同步入口与离线保护，不把脚本断言当作唯一依据。

| 场景 | 用户预期 | 实际观察 | 判定 |
|---|---|---|---|
| 新用户首次进入 | 首页/引导页能清楚开始，不被账号系统阻断 | 首屏展示“数学大冒险”和“开始冒险”；完成昵称后进入首页，主题和头像入口可见 | 通过 |
| Profile 账号区 | 未配置 Supabase 时不能出现不可完成的登录动作 | 当前环境未配置 Supabase；Profile 的账号区显示“当前版本未接入账号系统。”，无错误弹窗、无死按钮 | 通过 |
| 已有本地进度用户 | 本地星级和段位赛入口不因账号/同步状态消失 | 注入全题型进阶进度后，首页展示“挑战 新秀”，进入段位赛大厅后新秀挑战按钮在线可用 | 通过 |
| 离线段位赛门禁 | 离线时不能发起新挑战，但应保留进度可见性 | 切离线后大厅显示“当前离线，段位赛需要联网才能进行。恢复网络后可开始挑战。”，新秀挑战按钮禁用，段位列表仍可见 | 通过 |

## 体验判断

- 对未配置 Supabase 的本地环境，当前 UI 选择“安全降级”而不是暴露登录入口，符合普通用户预期。
- 离线段位赛的解释文案在操作点附近出现，用户能理解为什么挑战按钮不可点。
- 已有本地进度不会因为离线或未登录而从首页/段位赛大厅消失，这点对 v0.3 的“账号同步不应破坏本地进度体验”很关键。
