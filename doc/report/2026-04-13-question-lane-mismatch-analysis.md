# 闯关关卡主题与实际题目不匹配 — 分析报告

> **项目**: math-quest（数学大冒险）  
> **阶段**: Phase 1 游戏化重制  
> **分析日期**: 2026-04-13  
> **分析来源**: Code Review + 8 个生成器源码逐行审计  
> **状态**: 待专家评审 → 制定修复方案

---

## 1. 问题现象

用户在实际游玩中发现：

- **基础计算**的"加减主路"关卡中出现了乘法、除法和运算顺序题目
- **基础计算**的"乘除支路"关卡中出现了加法、减法题目
- 推测其他题型也存在类似的关卡标签与实际出题内容不匹配的问题

## 2. 根因分析

### 2.1 调用链路

```
campaign.ts 关卡定义
   ├── topicId (如 'mental-arithmetic')
   ├── difficulty (如 2)
   ├── laneLabel (如 '加减主路')  ← 仅用于UI显示
   └── stageLabel (如 '整数口算') ← 仅用于UI显示
         │
         ▼
store/index.ts → startCampaignSession()
   └── 只传递 topicId + difficulty
         │
         ▼
engine/index.ts → generateQuestion(topicId, difficulty)
   └── 查找 generators[topicId]，传入 { difficulty, id }
         │
         ▼
generators/*.ts → 内部按概率随机分发全部子题型
   └── 完全无视"当前是哪条路线 / 哪个阶段"
```

### 2.2 根因

**架构层缺失**：Lane / Stage 的语义标签（如"加减主路""分配律""添括号"）仅作为 UI 展示文字存在于 `campaign.ts` 中，**从未传递给出题引擎**。`generateQuestion()` 函数签名为：

```typescript
// src/engine/index.ts:30-33
export function generateQuestion(topicId: TopicId, difficulty: number): Question {
  const gen = generators[topicId];
  const d = Math.max(1, Math.min(10, Math.round(difficulty)));
  return gen({ difficulty: d, id: nanoid(10) });
}
```

只接受 `topicId` 和 `difficulty` 两个参数。每个生成器内部通过 `Math.random()` 从**全部子题型池**中随机抽取，导致无论玩家处于哪条路线，获得的题目分布完全相同。

### 2.3 关键数据

- `CampaignLane` 类型中无子题型过滤字段
- `GeneratorParams` 接口仅有 `difficulty` 和 `id`
- 8 个生成器均未接收路线/子题型过滤参数

## 3. 全量审查结果

### 3.1 A01 基础计算 (`mental-arithmetic`)

**生成器分布**（所有 difficulty ≤ 7）：
- 80% 单步口算：加减乘除各 ~25%（随机 `pickOperator`）
- 20% 运算顺序

| 路线 ID | 标称主题 | difficulty | 实际出题 | 匹配？ |
|---------|---------|-----------|---------|-------|
| S1-LA | 加减主路 | 2–4 | 加减乘除各25% + 运算顺序20% | **不匹配** — 出乘法、除法、运算顺序 |
| S1-LB | 乘除支路 | 2–3 | 同上 | **不匹配** — 出加法、减法、运算顺序 |

## 4. 汇总统计

| 题型 | 总路线数 | ✅ 匹配 | ⚠️ 部分匹配 | ❌ 不匹配 | 🔴 严重不匹配 |
|------|---------|--------|------------|---------|-------------|
| A01 基础计算 | 5 | 2 | 0 | 3 | 0 |
| A02 数感估算 | 8 | 2 | 0 | 6 | 0 |
| A03 竖式笔算 | 5 | 1 | 2 | 2 | 0 |
| A04 运算律 | 4 | 2 | 1 | 0 | **1** |
| A05 小数计算 | 6 | 2 | 0 | 4 | 0 |
| A06 括号变换 | 4 | 1 | 1 | 1 | **1** |
| A07 简便计算 | 7 | 2 | 0 | 5 | 0 |
| A08 方程移项 | 5 | 2 | 0 | 3 | 0 |
| **合计** | **44** | **14** | **4** | **24** | **2** |

**只有"综合"和"Boss"类路线是匹配的**（因为它们的设计意图就是混合出题）。**所有具有专项标签的路线全部不匹配或仅部分匹配。**

### 两个严重不匹配

1. **A04 运算律 S2"分配律"路线**：difficulty=4–5 在生成器中被归入 d≤5 分支，该分支**完全不生成分配律题**（分配律仅在 d>5 时生成）。导致标签为"分配律"的路线里**永远不会出现分配律题目**。

2. **A06 括号变换 S1"添括号"路线**：difficulty=2–4 在生成器中被归入 d≤5 分支，该分支**只生成去括号题**（60% 去+、40% 去-）。`generateAddBracket` 函数仅在 d>5 时被调用。导致标签为"添括号"的路线里出的**全是去括号题**。

## 5. 问题定性

| 维度 | 评估 |
|------|------|
| 严重程度 | **高** — 核心教学逻辑错误，玩家无法通过路线标签进行针对性练习 |
| 影响范围 | 44 条路线中 30 条（68%）受影响 |
| 问题性质 | 架构层缺失（系统性问题），非个别 Bug |
| 用户感知 | 玩家可直接感知到题目与关卡主题不对应 |

## 6. 修复方向（待评审确认）

修复需要跨 4 个层级的协调改动：

### 层级 1：数据层 (`campaign.ts`)
- 为 `CampaignLane`（或 `CampaignLevel`）新增 `subtypeFilter` 字段
- 定义每条路线允许的子题型枚举

### 层级 2：接口层 (`engine/index.ts`)
- `generateQuestion()` 增加可选的 `subtypeFilter` 参数
- `GeneratorParams` 接口扩展

### 层级 3：生成器层 (`generators/*.ts` × 8 个文件)
- 每个生成器增加按 `subtypeFilter` 过滤子题型的逻辑
- 在过滤后的子集内维持合理的概率分布

### 层级 4：调用层 (`store/index.ts`)
- `startCampaignSession` 从关卡定义读取 `subtypeFilter` 并传给 `generateQuestion`

### 涉及文件清单

| 文件 | 改动类型 |
|------|---------|
| `src/types/gamification.ts` | 新增 `subtypeFilter` 类型定义 |
| `src/constants/campaign.ts` | 44 条路线补充 `subtypeFilter` 配置 |
| `src/engine/index.ts` | 修改 `generateQuestion` / `GeneratorParams` |
| `src/engine/generators/mental-arithmetic.ts` | 增加过滤逻辑 |
| `src/engine/generators/number-sense.ts` | 增加过滤逻辑 |

| `src/engine/generators/operation-laws.ts` | 增加过滤逻辑 |
| `src/engine/generators/decimal-ops.ts` | 增加过滤逻辑 |
| `src/engine/generators/bracket-ops.ts` | 增加过滤逻辑 |
