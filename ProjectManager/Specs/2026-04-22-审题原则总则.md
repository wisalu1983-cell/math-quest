# 审题原则总则

> 版本：v1.0 | 创建：2026-04-22  
> 状态：**生效**  
> 适用范围：A01–A08 所有题型；作为 `4-2`/`4-3`/`4-4` 子计划的设计依据  
> 权威依据：Polya 四步法（1945）·中国小学数学审题教研文献

---

## 一、目的

本文件回答"审题能力是什么、哪些题型应该承载审题训练、如何设计题目来实现训练"。

**设计意图**：审题训练不是独立目标，而是在计算练习过程中**潜移默化**发生——学生感知到的是"做题"，审题能力在此过程中被自然调动和强化。

---

## 二、审题步骤定义

从学生开始看题到确定解题思路，共四步：

| 步骤 | 名称 | 含义 |
|---|---|---|
| **A** | 明确问题 | 搞清楚最终要解决什么，这道题要我产出什么答案 |
| **B** | 提取条件 | 找出题目给出的已知信息（数据、约束、情景） |
| **B'** | 建立数量关系 | 搭建"已知→未知"的数学桥梁；核心难点，学生最常卡住的地方 |
| **C/D** | 确定策略 | 判断题型 + 选择方法（两者合并：B'清晰后几乎自动得出） |

**说明**：
- B'（建立数量关系）≠ B（提取条件）。B 是"我手里有什么"，B' 是"这些怎么组合成答案"。学生条件都找到了但做不出题，通常是 B' 失败，而非 B 失败。
- 验证结果（回验）不属于审题范围，不在本总则讨论。
- 权威佐证：Polya 的"理解问题"阶段明确要求回答"条件与未知量之间的关系是什么"（= B'）；中国小学教研文献把"分析数量关系"列为审题核心步骤。

---

## 三、两个训练层次

| 层次 | 名称 | 描述 |
|---|---|---|
| **基础** | 正向审题 | 顺畅走完 A→B→B'→C/D：读懂情景 → 找到条件 → 建立关系 → 选对方法 |
| **进阶** | 对抗审题 | 在有干扰、误导、陷阱的题目里，识别噪音，找到真正有用的信息，走到正确路径 |

两个层次不是独立训练项，而是**随难度档位自然分配**：低档主要走正向，高档主要走对抗。

## 四、双机制设计原则

### 机制一：语义决定解法（正向审题 · 低档/基础关卡）

**定义**：题干的语义情景是学生得出解法的唯一路径；学生必须真正读懂情景才能判断用哪种方法。

**判断标准**：把方法名从 prompt 里删掉后，题目仍然完整可解，且不同学生对情景的不同理解会导致不同的方法选择——这道题才有审题价值。

**正例**：
```
买 8.3 米的绳子，商店只能整米卖。至少要买几米？
```
→ 学生必须读"至少"才能判断用进一法；题目没有说"用进一法"。

**反例**（审题价值清零）：
```
用进一法计算：买 8.3 米的绳子，商店只能整米卖，至少要买几米？
```
→ 方法已暴露，A/B'/C/D 全部跳过。

**语义信号词规律**（进一/去尾场景）：
- `至少`、`还差`、`不够就补` → 进一法（向上取整）
- `最多`、`最大整份`、`不够一份就舍` → 去尾法（向下取整）

---

### 机制二：陷阱倒逼精读（对抗审题 · 高档/进阶关卡）

**定义**：题目在结构或信息里埋有陷阱，粗心或依赖惯性思维的学生会掉进去；仔细逐项阅读能识破陷阱。

**判断标准**：
- ✅ **合法陷阱**：仔细读题能发现，属于精读能力的门槛
- ❌ **不合法陷阱**：题目语义本身歧义，或需要学生不知道的知识才能辨别——这是出题质量问题，不是审题训练

**典型合法陷阱类型**：

| 题型 | 陷阱类型 | 示例 |
|---|---|---|
| A04 运算律 | 分配律误用（`a×(b+c)` 误认为可分配但 b/c 是小数时） | distributive-trap |
| A06 括号 | 减号后括号去除变号遗漏 | 括号前是减号时符号全部反转 |
| A07 简便 | 看似能简便实则不能 | 形似结合律但不满足条件的算式 |
| A08 方程 | T1：减号后 x 项移位漏变号 | `a - bx = c` → 移项后 `-bx` 变号遗漏 |
| A08 方程 | T4：两边含 x 时移项方向混淆 | `ax + b = cx + d` 交叉移项 |

## 五、适用题型矩阵

**仅适用题型**（天然需要审题、题目设计应同时承载审题训练）：

| 题型 | 适用机制 | 主训练步骤 | 说明 |
|---|---|---|---|
| A02 进一/去尾情景题 | **机制一** | A + C/D | "至少/最多"的语义直接决定取整方向 |
| A02 估算方向判断 | **机制一** | B' | 估算偏大/偏小的方向需建立偏差分析 |
| A02 逆向四舍五入 | **机制一** | B' | 从结果逆推原数范围，数量关系须逆向建立 |
| A04 错误诊断/反例辨析 | **机制二** | A + B | 需明确"正确"的标准才能识别哪步出错 |
| A06 错误诊断 | **机制二** | B | 精读每一步变号是否正确 |
| A07 辨析哪个能简便（低档）| **机制一** | C/D | 判断算式结构是否满足简便条件 |
| A07 错误诊断（高档）| **机制二** | A + B | 先知道正确简便步骤，再找错误位置 |
| A08 陷阱题 T1–T4 | **机制二** | B | 精读每一项的符号和位置 |

## 六、不适用题型说明

以下题型**不强行加入审题设计**——它们的训练目标是计算能力本身，不是审题能力：

| 题型 | 原因 |
|---|---|
| A01 基础口算 | 纯算式，A/B'/C/D 全部显然，无审题空间 |
| A03 竖式笔算 | 纯算式，计算过程是唯一训练目标 |
| A05 小数性质 | 主要为概念理解和计算，极少涉及审题判断 |

**原则**：排除 = 承认"这个题型的目标是练计算，不是练审题"，不是遗漏。

## 七、A02 进一/去尾情景题专项规范

> A02 是当前工具中**唯一有文字情景的题型**，也是机制一最重要的载体。

### 7.1 情景题的审题价值来源

学生必须通过情景语义主动判断取整方向，而非依赖题干里的方法名：

- 情景包含"容纳/运送/装满"逻辑 → 不够就得多一个 → 进一法
- 情景包含"分配/截取/制作"逻辑 → 只能取完整份 → 去尾法

这是**A 步骤（明确问题）和 C/D（确定策略）**之间的直接映射训练——理解问题本身就决定了方法。

### 7.2 题目设计要求

1. **prompt 不得出现方法名**（"进一法"/"去尾法"/"向上取整"/"向下取整"）
2. 情景必须有明确的现实约束使学生自行判断方向
3. 答案必须是整数（除法有余数，余数决定取整方向）
4. 情景难度适中：单步除法，数字不超过五年级心算范围
