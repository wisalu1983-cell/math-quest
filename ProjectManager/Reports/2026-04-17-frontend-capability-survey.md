## 二、详细结论

### 能力 1：书面分数渲染（KaTeX）

- **意外发现**：`package.json` 已声明 `katex` 和 `@types/katex`，但整个 `src/` 代码库**从未引用**。
- 题干字段 `Question.promptLatex?: string`（`types/index.ts:40-41`）**已预留但未被 Practice 页消费**；例如 `equation-transpose.ts:90-92` 已经往里写 LaTeX，UI 却只渲染 `prompt`。
- **落地只需**：
  1. 在 Practice 页引入 `katex.min.css`
  2. 封装一个小组件 `<MathText />`，优先读 `promptLatex`
  3. 生成器侧在需要的地方填 `promptLatex`

**结论**：零依赖成本，几乎是"接线"级改动。

### 能力 2：多选 MC

- 当前 MC UI 是 `Practice.tsx:218-234` 内联渲染，状态为 `selectedOption: string | null`（单选）。
- 类型层：
  - `Solution.answer` 为 `number | string`（`types/index.ts:143-147`）——**无法表达多正确项**
  - `QuestionAttempt.userAnswer` 也是单字符串
- 改造面清晰：
  1. 在 `QuestionData` 或题型枚举里增加 `multiSelect: boolean` 标记
  2. `Solution.answer` 允许 `string[]` 或约定序列化格式（如 `"A,B,C"` 排序后比较）
  3. Practice 页改 checkbox UI + 集合相等判分
  4. 错题本/SessionDetail 的展示逻辑同步

**结论**：改动范围可控，先做类型升级即可。

### 能力 3：多步填空

- 当前**没有**任何 `fill-blank` 或 `multi-blank` 题型。
- 接近的能力是 `VerticalCalcBoard` + `DecimalTrainingGrid`，但它们是"竖式内部训练格"，**最终仍以单一值提交**。
- 类型里虽预留了 `'multi-step-input'` 和 `'expression-select'`（`types/index.ts:31-32`），但**没有任何生成器使用，Practice 也没有分支**。
- 改造要点：
  1. `Solution.answer` 支持数组或定义槽位结构
  2. `submitAnswer` 引入多值比较路径（按序/集合）
  3. 新增 `<MultiBlankInput />` 组件
  4. 生成器产出带槽位的 prompt（模板字符串 + 每空标准答案）

**结论**：有一定架构成本，但可与多选 MC 共享"答案容器升级"的底层改造。

### 能力 4：表达式输入 + 数学等价验证 ⚠️

- **`package.json` 中没有任何 CAS 库**（mathjs / mathsteps / algebra.js / nerdamer 都没有）。
- 现有判分逻辑在 `store/index.ts:161-183`：仅做字符串归一化（去空白、去省略号、去小数尾零）后**完全相等比较**；`number-sense.estimate` 子题型有 `acceptedAnswers` 数组机制，但也只是枚举允许值。
- 库选型建议：
  - **首选 mathjs**：TS 友好、社区成熟、支持 `simplify / compare` 思路、按需 tree-shake
  - **algebra.js / nerdamer**：符号代数更强，但 TS 体验与维护活跃度需专项评估
- **中文/全角处理**：无论用哪个库，都要在进库之前预处理 `．→.`、`，` 语境判断等

**决策分岔**：

| 路径 | 说明 | 适用场景 |
|------|------|---------|
| A：引入 mathjs | 可真正做代数等价，支持 `6x-2x=13+7` 与 `4x=20` 等价 | A06/A07/A08 完整实现 v2.1 规格 |
| B：降级为规则比较 | 生成器预先给出"标准规范式"，学生输入按固定规则归一化后比字符串 | 同题型范围收窄，只支持单一变形路径；题目出法受限 |
| C：混合方案 | A06 去/添括号用规则比较（允许项顺序可变但结构固定）；A07/A08 用 mathjs | 折中 |

## 五、降级方案清单（若不引入 mathjs）

| 能力 | 降级做法 |
|------|---------|
| A06 去/添括号填写式子 | 生成器同时产出"规范答案"和"等价项 Set"；学生输入按空白和运算符统一归一化，然后比较项集合与符号 |
| A07 简便变形 | 退化为 MC 选变形式（选项里藏经典错误），放弃"自由写变形"路径 |
| A08 移项填空 | 用"分步锁定模板"——每步给出最小可填部分（如"左边 = ___，右边 = ___"），每一步单独规则验证 |

降级方案能让规格 80% 落地，但会损失"开放写"的训练价值——学生如果只练 MC，变形能力仍然弱于自己写。
