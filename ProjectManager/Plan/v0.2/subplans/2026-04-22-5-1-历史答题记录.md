## 二、设计决策（方案设计阶段已完成）

### 2.1 数据架构

**两 key 分离，职责互不交叉**：

| Key | 用途 | 改动 |
|---|---|---|
| `mq_sessions` | 游戏运行时（段位赛刷新恢复等） | **不动** |
| `mq_history` | 历史记录 UI 数据源 | **新增** |

`mq_sessions` 目前同时承担"游戏状态持久化"职责（`resumeRankMatchGame` 从中读取完整 `rankQuestionQueue`），不可改为摘要。

### 2.2 数据类型

```ts
// 新增于 src/types/index.ts 或独立文件

interface HistoryRecord {
  id: string;                              // 同 PracticeSession.id
  userId: string;
  sessionMode: GameSessionMode;
  startedAt: number;                       // Unix ms，ISO 8601 兼容
  endedAt?: number;                        // 未完成局无此字段
  completed: boolean;
  result: 'win' | 'lose' | 'incomplete';  // 写入时计算，不从 UI 派生
  topicId: TopicId;                        // rank-match 取 primaryTopics[0] 兼容占位
  rankMatchMeta?: {
    primaryTopics: TopicId[];              // 段位赛模式标签展示用
  };
  questions: HistoryQuestionRecord[];
}

interface HistoryQuestionRecord {
  prompt: string;
  userAnswer: string;
  correctAnswer: string;   // solution.answer 转 string
  correct: boolean;
  timeMs: number;
}
```

### 2.3 result 计算规则（三种模式统一）

| 条件 | result |
|---|---|
| `completed === false` | `'incomplete'` |
| `completed === true && heartsRemaining > 0` | `'win'` |
| `completed === true && heartsRemaining === 0` | `'lose'` |

此规则对闯关、进阶、段位赛单局均适用。

### 2.4 写入时机

- `endSession()`：`completed: true`，写入 `mq_history`
- `abandonSession()`：`completed: false`，写入 `mq_history`

`cancelRankMatchSession()` / `suspendRankMatchSession()` 不写（段位赛赛事级操作，非普通答题结束）。

### 2.5 存档上限

`mq_history` 不设硬上限。摘要体积约 1KB/局，4MB 可存约 4000 局。未来接入云端后，移除本地上限即可，数据结构无需变更。

### 2.6 历史数据迁移

不迁移。Phase 5 上线前的局次存于 `mq_sessions`，不回填 `mq_history`。历史记录页只展示上线后产生的新数据。

### 2.7 存档版本号

不递增 `CURRENT_VERSION`。新增 key 读不到时返回空数组，旧存档零影响。

## 三、UI 设计

### 3.1 底部导航

`BottomNav` 中 `progress` tab 改名为"记录"（label 文案改，activeTab 标识符可保持 `'progress'` 或同步改为 `'history'`，视实施方便定）。

### 3.2 Progress.tsx 改为记录首页（合并页）

底部"记录" Tab 首达页改为 `Progress.tsx`，直接合并原"记录总览页"与"历史列表页"，减少一级页面跳转。

头部概览卡只保留：

- 标题："概览信息"
- 4 个统计项：累计答题 / 总正确率 / 已完成局数 / 胜利占比

界面约束：

- 去掉 `Overview` 英文眉题、说明文案、总场次卡片、"进入练习记录"按钮
- 4 个统计项在卡片内使用**单行信息布局**（数值 + 标签同一行），用于压缩头部高度
- 去掉原 `Progress.tsx` 中与历史列表重复分层的信息块（如三模式计数卡、Latest 最近一局）
- 头部概览卡下方直接接历史记录列表，不再先进入二级列表页

### 3.3 历史列表（History.tsx / 列表模块）字段与工具条

| 字段 | 显示规则 |
|---|---|
| 模式标签 | 闯关 / 进阶 / 段位赛（按 `sessionMode` 映射） |
| 日期 | `startedAt` 格式化 |
| 总体结果 | 胜利 / 失败 / 未完成（按 `result` 映射） |
| 正确率 | `correct/total`（`questions` 派生） |
| 耗时 | `endedAt - startedAt`；`completed === false` 显示 `—` |

工具条约束：

- 模式筛选：全部 / 闯关 / 进阶 / 段位赛，常驻展示
- 时间筛选：单独按钮，点击后打开浮窗
- 时间筛选浮窗：
  - 提供近 X 天快捷项（至少覆盖近 7 天 / 近 30 天 / 近 90 天）
  - 提供日历控件，支持选择某一天或明确的日期范围
- 排序：使用**同一个按钮**切换时间倒序 / 正序，不拆成两个按钮

结构约束：

- 一级入口由 `Progress.tsx` 承接；`History.tsx` 若保留，仅作为兼容页或内部复用壳，不再承接主导航首达
- 历史列表紧接概览卡，不额外插入"进入历史记录"中间层

### 3.4 SessionDetail.tsx 详情页

每题展示：
- 题干（`prompt`）
- 对/错标记
- 用户答案（答错时展示）
- 正确答案
- 单题耗时（`timeMs`）

去掉：`heartsRemaining`、`solution.explanation`（explanation 属于错题功能，不在本计划范围）

顶部汇总卡片：模式、日期、正确率、做题数/正确数、总耗时（或"—"）。

返回路径：优先回到合并后的记录首页（`progress`）。

## 四、实施步骤

2. 在 `KEYS` 中追加 `history: () => \`\${keyPrefix}history\``
3. 新增 `repository.getHistory(): HistoryRecord[]`
4. 新增 `repository.saveHistoryRecord(record: HistoryRecord): void`（push，不设上限）
5. `clearAll()` 中追加 `localStorage.removeItem(KEYS.history())`（dev namespace 路径已覆盖）

### Step 2：写入逻辑（store）

**文件**：`src/store/index.ts`

在 `endSession()` 和 `abandonSession()` 中，构建 `HistoryRecord` 并调用 `repository.saveHistoryRecord()`：

```ts
// endSession 结尾 / abandonSession 结尾追加
const historyRecord: HistoryRecord = {
  id: session.id,
  userId: session.userId,
  sessionMode: session.sessionMode,
  startedAt: session.startedAt,
  endedAt: session.endedAt,        // 已由各路径写入
  completed: session.completed,
  result: !session.completed
    ? 'incomplete'
    : hearts > 0 ? 'win' : 'lose',
  topicId: session.topicId,
  rankMatchMeta: session.rankMatchMeta
    ? { primaryTopics: session.rankMatchMeta.primaryTopics }
    : undefined,
  questions: session.questions.map(q => ({
    prompt: q.question.prompt,
    userAnswer: q.userAnswer,
    correctAnswer: String(q.question.solution.answer),
    correct: q.correct,
    timeMs: q.timeMs,
  })),
};
repository.saveHistoryRecord(historyRecord);
```

> **注意**：`abandonSession` 中 `session.endedAt` 已在 `saveSession` 时写入 `Date.now()`，但 `hearts` 在 abandon 路径对应的是 `get().hearts`，计算 `result` 时需确认取值来源与 `endSession` 路径一致。

### Step 3：UI 改造

**文件**：`src/pages/History.tsx`、`src/pages/SessionDetail.tsx`、`src/pages/Progress.tsx`、`src/components/BottomNav.tsx`（若需改 label）

### Step 4：单测

**新增测试覆盖**：
