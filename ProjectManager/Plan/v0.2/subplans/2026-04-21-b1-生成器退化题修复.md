### 根因

`generateRoundBasic` 和 `generateFloorCeilBasic` 可随机生成答案等于输入的退化题：

- **round**：`num = randInt(100, numMax)`，若 `num % place === 0`（如 680 mod 10 = 0），则 `answer = num`
- **floor-ceil（整数取整路径）**：`num = randInt(11, 999) / 10`，若 randInt 为 10 的倍数（如 110 → num=11.0），`Math.floor(11.0) = 11 = num`
- **floor-ceil（十分位路径，dp=2）**：同理，randInt 为 10 的倍数时 num 的十分位后无有效数字

## 二、设计决策

1. **Retry 上限 50 次**：与同文件 `generateEstimateBasic` 的 `attempt < 50` 先例一致
2. **兜底策略**：50 次全退化属极端情况（概率 < 10⁻¹⁰），`num += 1` 强制破退化，不做复杂逻辑
3. **不改外部签名**：调用方无感知

### 退化判定（伪代码）

```ts
// generateRoundBasic
for (let attempt = 0; attempt < 50; attempt++) {
  num = 随机生成();
  answer = Math.floor(num / place + 0.5) * place;
  if (answer !== num) break;
  if (attempt === 49) num += 1;
}

// generateFloorCeilBasic —— toInteger 路径
for (let attempt = 0; attempt < 50; attempt++) {
  num = randInt(11, maxRange) / factor;
  if (num % 1 !== 0) break;
  if (attempt === 49) num += 0.1;
}

// generateFloorCeilBasic —— 十分位路径（dp=2）
// 退化条件：小数第二位为 0（num*100 % 10 === 0）
// 检查：Math.round(num * 100) % 10 !== 0
```
