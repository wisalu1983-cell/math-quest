### 根因

`Practice.tsx` 题干 `<h2>` 按总字符数断点调整字号（>25→20px / >18→24px / ≤18→32px），不区分中文全角与 ASCII 半角宽度差异。  
在 375px 视口下，部分中长题干（如"不计算，比较大小: 4.5 × 3.2 ○ 4.5 × 1.2"，约 30 字符）会在算式中间折行。

### 目标视口

**375px**（iPhone SE / 13 mini）：外层 `px-4`×2 + 卡片 `px-5`×2 = 可用内容宽 **303px**。

---

## 二、设计决策

1. **权重字符数**：中文字符（CJK + 全角标点）权重 1.0，ASCII/数字/运算符权重 0.6，比"总字符数"更准确地估算渲染宽度
2. **字号公式**：`fontSize = clamp(14, floor(303 / wLen), 32)`（px）
3. **移除 `break-words`**：改为纯 `whitespace-normal`（允许在空格处换行，但不强制字符级折行），防止算式被切断
4. **字号下限 14px**：高于规格下限 11px，为极长题干保留安全边距

### 字号示意

| 题干示例 | wLen | fontSize |
|---|---|---|
| "56 + 34" | 4.2 | 32px（上限） |
| "将 680 四舍五入到十位" | 10.8 | 28px |
| "估算 492 + 435，结果取整十数" | 15.6 | 19px |
| "不计算，比较大小: 4.5 × 3.2 ○ 4.5 × 1.2" | 19.8 | 15px |

### 新增辅助函数（`Practice.tsx` 函数组件外）

```ts
function promptFontSize(prompt: string): number {
  const wLen = [...prompt].reduce(
    (acc, c) => acc + (/[\u4e00-\u9fff\uff00-\uffef，。？！：；]/.test(c) ? 1.0 : 0.6),
    0
  );
  return Math.max(14, Math.min(32, Math.floor(303 / wLen)));
}
```
