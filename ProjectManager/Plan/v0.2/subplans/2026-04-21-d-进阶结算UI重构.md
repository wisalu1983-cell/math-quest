## 问题描述

`SessionSummary.tsx` 的 `AdvanceSummary` 组件当前只做静态展示，玩家无法感知"这局 +N 心推进了多少进度"。

诊断报告：[`Reports/2026-04-21-phase-2-diagnosis.md`](../../Reports/2026-04-21-phase-2-diagnosis.md) §三

### v1 方案被否的原因

v1 做了简单的进度条 CSS transition（before→after 宽度变化）+ 文字对比标注。问题：**心和星级进度之间的因果关系仍然是文字叙述而非视觉呈现**。用户期望的是心"注入"到进度条的动画过程，让玩家直觉感知"剩下的心变成了星级经验"。

---

## 数据层（无需改动）

| 变量 | 来源 | 含义 |
|---|---|---|
| `heartsAccBefore` | `heartsAccAfter - heartsEarned` | 本局前累计心数 |
| `heartsAccAfter` | `progress.heartsAccumulated` | 本局后累计心数 |
| `starsBefore` | `getStars(heartsAccBefore, cap)` | 本局前星级 |
| `starsAfter` | `currentStars` | 本局后星级 |
| `leveled` | `starsAfter > starsBefore` | 是否升星 |
| `starProgressBefore` | `getStarProgress(max(0, heartsAccBefore), cap)` | 本局前星内进度 |
| `starProgress` | `getStarProgress(heartsAccAfter, cap)` | 本局后星内进度 |

### 每颗心的中间态计算

不能简单均分 `(starProgress - starProgressBefore) / heartsEarned`——跨星级时 before/after 的 starProgress 基准不同。

正确方法：逐颗计算真实累计心数对应的进度：

```ts
type HeartStep = {
  stars: number;           // 注入后的星级
  progress: number;        // 注入后的星内进度 (0~1)
  leveledUp: boolean;      // 这颗心是否触发了升星
};

const steps: HeartStep[] = [];
for (let i = 1; i <= heartsEarned; i++) {
  const acc = heartsAccBefore + i;
  const s = getStars(acc, cap);
  const p = getStarProgress(acc, cap);
  steps.push({ stars: s, progress: p, leveledUp: s > (steps[i-2]?.stars ?? starsBefore) });
}
```

### 约束确认

- 每局最多 +3 心，最小星段间距 6 心 → **单局最多跨 1 次星级边界**
- 不存在"一局升两星"的情况

## 交互方案 v2：心注入动效

### 核心理念

**叙事线**：你赢得了这些心 → 看着它们一颗颗注入进度条 → 进度在你眼前增长 → （如果升星）星星亮起来

用动画取代文字来传达"心 = 星级经验"这个核心机制。

### 布局调整

将原来分离的"心数投入"卡片和"星级进度"卡片合并为一个卡片，让心图标和进度条在同一个视觉容器内，动画路径更自然：

```
┌────────────────────────────────┐
│  Banner（通关/升星）            │
└────────────────────────────────┘

┌────────────────────────────────┐  ← 合并后的"进阶进度"卡片
│                                │
│     ★ ★ ★ ☆ ☆   (星星行)       │
│                                │
│     ❤️  ❤️  ❤️    (待注入心行)   │
│        ↓                       │
│  ▓▓▓▓▓▓▓░░░░░░░░ 42% → 3★    │
│                                │
│  每局最多+3❤️ · 积20❤️升1★      │
└────────────────────────────────┘

┌──────────┐ ┌──────────┐
│ 3/5 答对  │ │ 60% 正确 │   ← 答题统计（不变）
└──────────┘ └──────────┘

[ 再来一局 ▶ ]                    ← 操作按钮（不变）
[   回首页   ]
```

### 分镜脚本

#### Scene 1：落地入场（0 ~ 600ms）

| 时刻 | 元素 | 动作 |
|---|---|---|
| 0ms | Banner | 从上方淡入滑下（stagger-1，现有动效） |
| 200ms | 进阶进度卡片 | 淡入（stagger-2） |
| 200ms | 星星行 | 显示 `starsBefore` 个亮星（不是 starsAfter） |
| 200ms | 进度条 | 停在 `starProgressBefore` 位置 |
| 400ms | 心行 | `heartsEarned` 个心图标依次弹入（scale 0→1，间隔 100ms） |
| 400ms | 心行 | 入场后轻微上下浮动（pulse 动画），暗示"准备就绪" |

此刻画面静止，玩家看到：我赢了 N 颗心，当前进度在这里。

#### Scene 2：逐心注入（800ms 起）

**每颗心的注入周期约 600ms，心间间隔 200ms。**

单颗心注入动画分解：

| 阶段 | 时长 | 描述 |
|---|---|---|
| 起飞 | 0~100ms | 心图标从心行位置向上微弹（y -8px），同时开始缩小（scale 1→0.6） |
| 飞行 | 100~350ms | 心图标沿弧线向下飞向进度条头部位置（用 framer-motion `animate` 控制 x/y） |
| 着陆 | 350ms | 心图标到达进度条右端，opacity→0 消失 |
| 脉冲 | 350~400ms | 进度条右端发出一次短暂的亮色脉冲（box-shadow expand + fade） |
| 填充 | 350~600ms | 进度条宽度从当前值 transition 到该颗心对应的 `intermediateProgress`（ease-out） |

**三颗心的完整时序（以 heartsEarned=3、不升星为例）：**

| 时刻(ms) | 事件 |
|---|---|
| 800 | ❤️ #1 起飞，心行中 #1 开始飞离 |
| 1150 | ❤️ #1 着陆进度条，脉冲，进度条开始增长至 step[0].progress |
| 1400 | 进度条到达 step[0] 位置 |
| 1600 | ❤️ #2 起飞 |
| 1950 | ❤️ #2 着陆，进度条增长至 step[1].progress |
| 2200 | 进度条到达 step[1] 位置 |
| 2400 | ❤️ #3 起飞 |
| 2750 | ❤️ #3 着陆，进度条增长至 step[2].progress（最终值） |
| 3050 | 进度条到达最终位置 |
| 3400 | 对比文字淡入 |

#### Scene 2b：升星触发（仅 `leveled=true`）

当某颗心的 `step[i].leveledUp === true` 时，该颗心的注入周期内插入升星动画：

| 阶段 | 时长 | 描述 |
|---|---|---|
| 进度条填满 | 0~250ms | 进度条先增长至 100%，而非直接跳到新星段 |
| 闪光 | 250~450ms | 进度条整体闪烁一次（opacity 脉冲 1→0.5→1），背景色短暂变亮 |
| 星星点亮 | 450ms | 下一颗星从 ☆ 变 ★（scale 1→1.4→1 弹跳，~300ms） |
| Confetti | 500ms | `ConfettiEffect` 触发 |
| Banner 变色 | 500ms | Banner 背景切换为 warning 配色（bg-warning-lt），emoji 变 🌟 |
| 进度条重置 | 700ms | 进度条宽度瞬间归 0（无 transition），然后开始增长至新星段内的剩余进度 |
| 继续注入 | 900ms+ | 如果还有剩余心，继续后续心的注入 |

#### Scene 3：定格（全部注入完毕后 +300ms）

- 心行区域淡出或显示为"已全部注入"状态（三颗灰色空心轮廓）
- 进度条下方淡入最终数据：`进度 {before}% → {after}%`
- 简注文字静态显示：`每局最多 +3 ❤️ · 积 {nextThreshold} ❤️ 升 1★`

### heartsEarned = 0 的特殊处理

不播放任何注入动画。心行显示"本局未获得心"，进度条直接静态显示当前进度。Banner 保持"白练一局"样式。与现有行为一致。

### 升星 Banner 样式对照

| 条件 | Banner 背景 | emoji | 标题 |
|---|---|---|---|
| `leveled` | `bg-warning-lt border-warning` | 🌟 | 升星啦！ |
| `heartsEarned > 0 && !leveled` | `bg-success-lt border-success-mid` | 🎉/💪 | 练习完成！ |
| `heartsEarned === 0` | `bg-card border-border-2` | 😅 | 白练一局，继续加油！ |

注意：升星 Banner 变色发生在 Scene 2b 的 500ms 时刻（由注入动画触发），不是页面加载时。初始加载时 Banner 统一使用 `heartsEarned > 0` 的绿色样式，升星后才切换为黄色。

## 技术方案

### 依赖

- `framer-motion`：已在项目中（`ConfettiEffect` 在用），用于心飞行动画和入场动画
- `ConfettiEffect`：`{ active: boolean }` 接口，已确认

### 状态机

用一个 `injectionPhase` 状态驱动整个动画序列：

```ts
type InjectionPhase =
  | 'entering'       // Scene 1: 入场中
  | 'injecting'      // Scene 2: 正在逐心注入
  | 'leveling-up'    // Scene 2b: 升星动画中
  | 'done';          // Scene 3: 全部完成

const [phase, setPhase] = useState<InjectionPhase>('entering');
const [injectedCount, setInjectedCount] = useState(0); // 已注入几颗心
const [displayProgress, setDisplayProgress] = useState(starProgressBefore); // 当前进度条显示值
const [displayStars, setDisplayStars] = useState(starsBefore); // 当前显示的亮星数
const [showConfetti, setShowConfetti] = useState(false);
const [bannerStyle, setBannerStyle] = useState<'success' | 'warning' | 'neutral'>(
  heartsEarned > 0 ? 'success' : 'neutral'
);
```

### 动画编排

用 `useEffect` + `setTimeout` 链驱动（不用 requestAnimationFrame，因为是离散的阶段切换）：

```ts
useEffect(() => {
  if (heartsEarned === 0) { setPhase('done'); return; }

  // Scene 1 → Scene 2
  const enterTimer = setTimeout(() => {
    setPhase('injecting');
    injectNext(0); // 开始注入第一颗心
  }, 800);

  return () => clearTimeout(enterTimer);
}, []);

function injectNext(index: number) {
  if (index >= heartsEarned) {
    setPhase('done');
    return;
  }
  const step = steps[index];

  if (step.leveledUp) {
    // 先填满到 100%
    setDisplayProgress(1.0);
    setTimeout(() => {
      setPhase('leveling-up');
      setDisplayStars(step.stars);
      setShowConfetti(true);
      setBannerStyle('warning');
      setTimeout(() => {
        setDisplayProgress(0); // 瞬间归零（此时关闭 transition）
        requestAnimationFrame(() => {
          setDisplayProgress(step.progress); // 重新开启 transition，增长到新位置
          setInjectedCount(index + 1);
          setTimeout(() => injectNext(index + 1), 800);
        });
      }, 700);
    }, 400);
  } else {
    setDisplayProgress(step.progress);
    setInjectedCount(index + 1);
    setTimeout(() => injectNext(index + 1), 800);
  }
}
```

### 心飞行动画

每颗心用 `motion.div` 包裹。当 `injectedCount` 将要超过该心的 index 时，触发飞行动画：

```tsx
<motion.div
  animate={isFlying ? {
    y: [0, -8, progressBarOffsetY],   // 先微弹，再飞向进度条
    x: [0, 0, progressBarOffsetX],
    scale: [1, 0.6, 0.3],
    opacity: [1, 1, 0],
  } : { scale: [0, 1.2, 1] /* 入场弹入 */ }}
  transition={{ duration: 0.35, ease: 'easeIn' }}
/>
```

心的飞行目标位置：进度条的当前右端。由于心行和进度条在同一个卡片内，用相对偏移量即可（无需 `useRef` 精确测量，取 `y ≈ +60px, x ≈ 0` 的固定近似值，因为卡片布局固定）。

### 进度条 transition 控制

升星时需要进度条瞬间归零（无 transition），然后重新开启 transition 增长。用一个 `skipTransition` state 控制：

```tsx
<div
  className={`h-2.5 bg-warning rounded-full ${
    skipTransition ? '' : 'transition-[width] duration-500 ease-out'
  }`}
  style={{ width: `${Math.round(displayProgress * 100)}%` }}
/>
```

## 验收标准

1. `heartsEarned > 0` 时，心图标逐颗从心行飞入进度条，每颗心到达时进度条可见增长
2. 升星时：进度条先填满 → 闪光 → 新星点亮弹跳 → Confetti → Banner 变色 → 进度条归零后继续
3. `heartsEarned = 0` 时组件行为与改动前一致（静态展示，无动画）
4. 已满级时不显示进度条，显示"已达满级 🏆"
5. 简注文字正确显示当前星段所需心数
6. `vitest` 全绿，`npm run build` 无报错
