## 问题描述

`SessionSummary.tsx` 的 `AdvanceSummary` 组件当前只做静态展示，玩家无法感知"这局 +N 心推进了多少进度"。

诊断报告：[`Reports/2026-04-21-phase-2-diagnosis.md`](../../Reports/2026-04-21-phase-2-diagnosis.md) §三

## 数据现状（无需改动数据层）

| 变量 | 来源 | 含义 |
|---|---|---|
| `heartsAccBefore` | `heartsAccAfter - heartsEarned` | 本局前累计心数 |
| `heartsAccAfter` | `progress.heartsAccumulated` | 本局后累计心数 |
| `starsBefore` | `getStars(heartsAccBefore, cap)` | 本局前星级 |
| `starsAfter` | `currentStars` | 本局后星级 |
| `leveled` | `starsAfter > starsBefore` | 是否升星 |
| `starProgress` | `getStarProgress(heartsAccAfter, cap)` | 当前星内进度（after） |

缺：`starProgressBefore = getStarProgress(max(0, heartsAccBefore), cap)`，需在组件内补一行计算。

---

## 改动方案

### 1. 进度条 before → after 动效

**当前**：进度条宽度静止显示 `starProgress * 100%`。

**目标**：
- 初始渲染时宽度 = `starProgressBefore * 100%`
- delay 300ms 后 transition 到 `starProgress * 100%`（0.7s ease-out）
- 实现：`useState(false)` + `useEffect` 延迟 set；CSS `transition-[width] duration-700 delay-300`

### 2. 进度对比标注

在进度条下方（现有 `{Math.round(starProgress * 100)}% → {starsAfter + 1}★` 文字行）补充：

```
本局 +{heartsEarned} ❤️  |  进度 {Math.round(starProgressBefore*100)}% → {Math.round(starProgress*100)}%
```

### 3. 升星时专属 Banner + Confetti

**当前**：升星时仅在星级进度卡片右上角显示小徽章"✨ 升星！"

**目标**：
- 主 Banner 区（顶部 py-7 圆角卡片）升星时切换为专属样式（黄底 / warning 配色，emoji 用 🌟）
- 接入已有 `ConfettiEffect` 组件（确认其 props 后决定放置位置）
- 保留原有小徽章

### 4. 心→星简注文字

在星级进度卡片底部补一行说明：

```
每局最多 +3 ❤️  · 积 {nextThreshold} ❤️ 升 1★
```

`nextThreshold` = 下一星级门槛心数（从 `STAR_THRESHOLDS_3` / `STAR_THRESHOLDS_5` 推算，取当前星段所需增量）。

## 验收标准

1. `heartsEarned > 0` 时，进度条有从 before 到 after 的可见填入动效
2. 进度对比数字正确显示（before% → after%）
3. 升星时主 Banner 背景变为 warning 配色，ConfettiEffect 触发
4. `heartsEarned = 0` 时组件行为与改动前一致
5. `vitest` 全绿，`npm run build` 无报错
