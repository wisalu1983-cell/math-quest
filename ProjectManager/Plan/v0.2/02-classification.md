## 六类一览

### A. 题型教育设计层（先定原则，再改题）

| 子项 | 范围 | 依赖 |
|---|---|---|
| **A3** | 审题能力跨题型训练原则（**元级总则**）| 无（A 组内前置）|
| **A1** | 估算题目的与精度定义 | 弱依赖 A3 |
| **A2** | 基础技巧类题型统一排查（估算/数感等；口算笔算不在此列）| 弱依赖 A3 |
| **A4** | 逆向推理第 1 关作为 A3 的验证用例 | 强依赖 A3 |

### B. 生成器实现/约束

| 子项 | 范围 | 性质 |
|---|---|---|
| **B1** | 四舍五入/去尾/进一 禁止"结果=输入"退化题 | 规则明确，独立小修 |
| **B2** | 口算进阶 1–2🌟 仍出现个位×个位 | 需诊断定性（设计 vs 实现）|

### C. 关卡/难度曲线

| 子项 | 范围 | 性质 |
|---|---|---|
| **C1** | 同一行 / 同知识点关卡难度递进 | 需诊断扫描 |

### D. 进阶模式一局结算优化

| 子项 | 范围 | 性质 |
|---|---|---|
| **D** | 心↔星级进度关系的可视化；每局结算页星级变化 UI/动效 | 需诊断定性（设计未实现 vs 设计不到位）|

### E. UI/排版

| 子项 | 范围 | 工作量 |
|---|---|---|
| **E1** | 题干算式不折行；必要的最小自适应（容器宽度/字号/溢出处理）| 收敛范围，约 1~2 天级 |

### F. 新功能

| 子项 | 范围 | 依赖 |
|---|---|---|
| **F1** | 按题型/技巧匹配的方法 Tips 库 | 强依赖 A1/A2 产出的"题型-技巧"映射表 |
| **F2** | 历史答题记录（按模式；每局详细记录）| 本地存档，独立 |
| **F3** | 开发者工具栏（账号状态一键注入）| 独立；**最高杠杆** |

## 依赖关系图

```
─── 完全独立（可立即启动）───────
F3      B1      E1      F2

─── 需先诊断 ─────────────────
B2 ────┐
C1 ────┼──→ 诊断结论 ──→ 归"补实现 / 补规则 / 归入 A"
D  ────┘

─── A 组链路 ────────────────
A3 ──→ A1 ──┐
         A2 ──┼──→ F1 （Tips 库 = A 组"题型-技巧"映射的 UI 落地）
         A4 ──┘（作为 A3 的验证）
```

## 两点关键判断

1. **F1 不作为独立并行新功能**：它的内容核心就是 A 组产出的"技巧清单"；独立做会导致 A 组完成后需要重做 Tips。**→ F1 归在 A 组之后执行，作为 `4-4`。**
2. **F2 不再等账号系统**：可在本地存档基础上先做；账号系统日后接入时做迁移即可，遵守项目级"存档版本升级"原则（见 `CLAUDE.md` §非显然约束）。

| Phase | 承载的子项 |

| **3** | Phase 2 结论里定性为"实现/规则问题"的修复 |
| **4** | A3 · A1 · A2 · A4 · F1 |
| **5** | F2 |
