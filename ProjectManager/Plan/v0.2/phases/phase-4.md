## δ-1：A3 审题原则总则

**必须回答的问题**（启动 4 步流程时写入"预期效果"）：

- 审题能力的本质是什么？（读题/找信息/判断条件/选工具/回验结果 ······ 的哪些环节？）
- 五年级学生应掌握到哪一层？
- 哪些题型天然承载审题训练？哪些不承载？
- 审题训练在题干层面有哪些可执行的设计手法？（语义变体 / 条件冗余 / 隐藏条件 / 多步嵌套 / 情景包装 ······）

## δ-2：A1 估算 + A2 基础技巧类题型排查

### A1 估算题

**必须回答的问题**：

- 估算能力要服务什么场景？（心算验算 / 合理性判断 / 大数量级判断 / 购物找零 ······）
- 五年级的估算精度应到什么层级？（整百 / 整十 / 有效数字）
- 当前生成器里的估算题（如 `492+435` 估整十位）为什么"估完和精确值差不多"？是题型设计问题还是参数配置问题？

### A2 基础技巧类题型排查

**排查范围**：估算、数感（四舍五入 / 去尾 / 进一 / 数位理解 / 数的组成）、逆向推理等

**排除范围**：口算、笔算（用户明确不在此列）

**必须产出**：

- 一张"题型 × 能力训练目的"映射表
- 每个题型"是否需要引入应用场景（而不是机械操作训练）"的判断
- 对需要重设计的题型：用户预期的"应用情景"举例

## δ-3：A4 逆向推理验证用例

用"逆向推理第 1 关"作为 A3 原则的**回验**：按 A3 的审题原则审视当前题干机械化问题，提出多样化方案。

这是**验证步骤**，不是独立需求——若 A3 原则能正确指导 A4 的重设计，说明 A3 站得住。

## δ-4：F1 方法 Tips 库落地

F1 的内容核心 = A1/A2 产出的"题型-技巧"映射表的 UI 表达。

设计原则（已在 `01-feedback-catalog.md` §澄清边界固化）：

- **按题型/技巧匹配**，不是每题一套专用 tips
- **题干阶段**展示，不是答题后再给
- 当学生"需要使用某技巧却可能不熟练"时触发
