**已产出**：[`../../../Specs/2026-04-22-审题原则总则.md`](../../../Specs/2026-04-22-审题原则总则.md)

**必须回答的问题**（启动 4 步流程时写入"预期效果"）：

- 审题能力的本质是什么？（读题/找信息/判断条件/选工具/回验结果 ······ 的哪些环节？）
- 五年级学生应掌握到哪一层？
- 哪些题型天然承载审题训练？哪些不承载？
- 审题训练在题干层面有哪些可执行的设计手法？（语义变体 / 条件冗余 / 隐藏条件 / 多步嵌套 / 情景包装 ······）

## `4-2`：A1 估算 + A2 基础技巧类题型排查

**已产出**：[`../../../Specs/2026-04-22-估算能力与基础技巧类排查.md`](../../../Specs/2026-04-22-估算能力与基础技巧类排查.md)  
**实施子项**：[`../subplans/2026-04-22-4-2a-estimate-basic重设计.md`](../subplans/2026-04-22-4-2a-estimate-basic重设计.md) · [`../subplans/2026-04-22-4-2b-floor-ceil重构.md`](../subplans/2026-04-22-4-2b-floor-ceil重构.md)

### A1 估算题

**必须回答的问题**：

- 估算能力要服务什么场景？（心算验算 / 合理性判断 / 大数量级判断 / 购物找零 ······）
- 五年级的估算精度应到什么层级？（整百 / 整十 / 有效数字）
- 当前生成器里的估算题（如 `492+435` 估整十位）为什么"估完和精确值差不多"？是题型设计问题还是参数配置问题？

### A2 基础技巧类题型排查

**排查范围**：估算、数感（四舍五入 / 去尾 / 进一 / 数位理解 / 数的组成）、逆向推理等

**排除范围**：口算、笔算（用户明确不在此列）

**必须产出**：

- 一张"题型 × 能力训练目的"映射表
- 每个题型"是否需要引入应用场景（而不是机械操作训练）"的判断
- 对需要重设计的题型：用户预期的"应用情景"举例
