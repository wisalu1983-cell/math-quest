## 五个 Phase 一览

| Phase | 名称 | 主产出 | 启动条件 | 收尾条件 |
|---|---|---|---|---|
| **α** | 效率基建 + 低成本修复 | F3 工具栏可用；B1 / E1 上线 | 主线整体方向确认 | F3 可用 + B1/E1 合并入仓 + 通过回归 |
| **β** | 三项合并短诊断 | 联合诊断报告（B2 / C1 / D）| α 完成 | 报告落盘；三项各有归位建议 |
| **γ** | 诊断结论执行 | β 结论里定性为"实现/规则问题"的修复 | β 完成 | 涉及项通过回归；归 A 组的项已登记到 δ |
| **δ** | 题型教育设计重梳理 | A3 总则 + A1/A2 重设计 + A4 验证 + F1 落地 | δ-1 可与 α/β 并行；δ-2/δ-3 需 A3 通过 | A 组规格/实现两端收敛；F1 Tips 库上线 |
| **ε** | 历史答题记录 | F2 本地版 | α 完成后任意时机 | F2 覆盖闯关/进阶/段位三模式 |

## 时序示意

```
时间轴 →
 Phase α   [α-1 F3]──[α-2 B1+E1]
 Phase β                              [三项合并诊断]
 Phase γ                                          [结论执行 1~3 子计划]
 Phase δ   [δ-1 A3 起草，可与 α/β 并行] ────── [δ-2 A1+A2] ── [δ-3 A4] ── [δ-4 F1]
 Phase ε                                                          [F2，可在 γ/δ 期间穿插]
```

## 并行可能性

- **δ-1（A3 起草）**：纯设计讨论、不动代码，可与 α/β 并行启动，不占用工程带宽
- **Phase ε（F2）**：本地存档独立，可穿插于 γ/δ 期间执行
- **α-1（F3）与 α-2（B1+E1）**：之间无硬依赖，按工程带宽决定串行或并行

## Phase 之间的硬依赖 vs 软依赖

| 关系 | 性质 | 说明 |
|---|---|---|
| α → β | **软依赖** | 不是 β 技术上必须等 α，而是 α 的 F3 工具栏能大幅降低 β 的诊断成本 |
| β → γ | **硬依赖** | γ 内容由 β 诊断报告决定 |
| α/β → δ-1 | **无依赖** | δ-1 纯文档起草可并行启动 |
| A3 → A1/A2/A4 | **硬依赖**（δ 内部）| A3 先通过，下游才开工 |
| A1/A2 → F1 | **硬依赖**（δ 内部）| F1 内容源于 A1/A2 的"题型-技巧"映射 |
| α → ε | **软依赖** | F3 工具栏能帮助 F2 造测试数据 |
