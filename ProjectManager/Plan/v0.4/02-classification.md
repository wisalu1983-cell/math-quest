```text
Phase 1：低风险修复
  ├─ A1 颜色
  ├─ A2 小数答案兼容
  ├─ A3 乘法竖式统一（多位整数模块优先）
  └─ B3 ISSUE-059

Phase 2：题目质量
  ├─ A4 竖式难度（含第四关少量 2位数×2位数桥接）
  ├─ B1 选项干扰项
  └─ B2 重复题目诊断

Phase 3：设计决策
  ├─ A5 进位格规则
  ├─ C1/C2 运算律 UX
  └─ C3 compare tip 补证

Phase 4：工程质量
  └─ D1 Practice 状态重置
```

## 关键边界

1. **先诊断再修重复题**：`BL-008` 可能是代码 bug，也可能是生成器设计缺口；不能在未诊断前直接定性。
2. **乘法竖式统一路线已确认**：先做多位整数乘法竖式模块，再让小数乘法复用该模块，最后加强小数点定位训练与答案等价判定。
3. **进位格规则不是纯技术修复**：它改变“过程正确性”和“最终答案正确性”的关系，必须先确认教学目标。
4. **选项题不是简单加数量**：扩到 3~4 个选项时，要加入概念混淆项，避免只增加无效干扰。
5. **Practice 重构必须行为等价**：Phase 4 只清理状态机制，不顺手改题型逻辑或页面流程。
