# Phase 5：Practice 工程质量

> 所属：v0.4-5
> 状态：📋 待确认是否保留在 v0.4
> 来源：`BL-004`

---

## 目标

把 Practice 答题页“切下一题时手动重置多份输入状态”的实现清理为统一机制，降低后续继续叠题型和交互时的维护成本。

本 Phase 是工程质量收尾项，不阻塞 Phase 2~4 的题目体验主线。若 v0.4 工作量超出预期，可延期到 v0.5。

## 范围

| 子项 | 内容 | 备注 |
|---|---|---|
| 5.1 | 梳理 Practice 当前状态重置点 | 只读调研 |
| 5.2 | 设计统一状态重置机制 | 必须保持用户行为等价 |
| 5.3 | 补回归测试 | 覆盖切题、答题、重试、不同题型输入 |

## 进入条件

- 用户确认 Phase 5 保留在 v0.4，而不是延期到 v0.5
- 先补或确认现有行为测试，避免重构改变体验

## 收尾条件

- 重构前后用户可见行为等价
- Practice 相关回归测试通过
- `npm test -- --run` 和 `npm run build` 通过
