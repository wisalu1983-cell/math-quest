# Phase 2：题目质量与生成器诊断

> 所属：v0.4-2
> 状态：📋 待开工
> 来源：`BL-005.4` / `BL-007` / `BL-008`

---

## 目标

修复题目本身的质量问题：难度要匹配进阶星级，选项题要降低猜测概率，重复题目要先查清真实原因再处理。

## 范围

| 子项 | 内容 | 备注 |
|---|---|---|
| 2.1 | A03 竖式难度梯度排查，剔除心算级题目混入 | 必查生成器难度、闯关关卡梯度和 `TOPIC_STAR_CAP` |
| 2.1a | 竖式笔算第四关加入少量 `2位数 × 2位数` 乘法 | 作为从一位乘数到多位乘法的桥接，建议占第四关 10%-20%，复用 Phase 1 多位乘法竖式板 |
| 2.2 | 选项题扩充到 3~4 个选项，加入概念混淆型干扰项 | 不只增加无意义选项 |
| 2.3 | 闯关 / 进阶 / 段位赛重复题目诊断 | 先判断是 bug 还是设计缺口 |

## 进入条件

- 读 `generator-redesign-v2.md`
- 读 `generator-difficulty-tiering-spec.md`
- 读 `gamification-phase2-advance-spec.md` 的 `TOPIC_STAR_CAP`
- 复核 `vertical-calc-S1-LB-L1/L2` 当前题目分布：第三关保持两位乘一位，第四关以三位乘一位 / 四位除一位为主，少量桥接 `2位数 × 2位数`
- 为 `BL-008` 确认诊断方式：脚本抽样 / 场景复现 / 两者结合

## 收尾条件

- 难度修复有抽样验证记录
- 第四关 `2位数 × 2位数` 出现比例受控，且不会替代第四关主体难度
- 选项题有覆盖 3~4 选项和概念混淆项的验证
- 重复题目有明确结论：已修 bug / 设计缺口转后续 / 无法复现并记录样本
- `npm test -- --run` 和 `npm run build` 通过
