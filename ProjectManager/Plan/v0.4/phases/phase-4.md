## 已确认设计决策

| 维度 | 决策 |
|---|---|
| 实现方案 | 采用 B 方案：新增 `vertical-calc-policy` 纯逻辑策略层，UI 只消费显示、跳格、提交、结果分类。 |
| 低档 | 显示过程格；期望值非 0 的过程格必填；答案对但过程格错仍未通过、扣心、进错题本。 |
| 低档 0 过程格 | 显示且不自动跳过；留空等同 0；用户可填 0，也可点击 / 触摸 / `Enter` / `Tab` 跳到下一个格子。 |
| 中档 | 显示可选过程格；答案正确即通过；过程格错误只在统一结果判定 UI 给当前题提示，不在竖式板提示，不入错题本 / 历史 / 统计。 |
| 高档 | 不显示进位/退位格。 |
| 填写完整 | 答案格为 `0-9`；加法过程格为 `0/1`；减法过程格为 `0/-1` 且单独 `-` 不完整；乘法过程格为 `0-8`。 |
| 提交拦截 | 低档非 0 过程格未填时不能提交，并显眼提示“请把进位/退位格填完整”。 |
