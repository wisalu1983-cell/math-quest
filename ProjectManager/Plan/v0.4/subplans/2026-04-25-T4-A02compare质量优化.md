## 一、问题定义

Phase 3 预研最初把 A02 高档 compare 的 `对 / 错` 二选一识别为“猜题形态”风险。经过本轮决策，用户确认：A02 高档 compare 的判断题应保留 `对 / 错` 二选一。T4 因此从“消灭二选一”改为“compare 质量优化”。

真正要解决的是：d=7 一步比较太浅，d=8 概念判断题池偏小、重复感明显，且 explanation 需要更像教学反馈而不是只报答案。

## 二、已确认决策

- 母题型：A02 `number-sense`。
- 子题型：`compare`。
- T4 是质量优化任务，不是题型形态替换。
- `d=7` 和 `d=8` 同等纳入。
- 保留 `d=8` 的 `对 / 错` 二选一。
- 保持能力分工：
  - `d=4-6`：基础 / 中档一步 compare。
  - `d=7`：二步结构 compare / 误导性 compare。
  - `d=8`：概念判断 compare。
- d=7 旧一步模板降到 `d=6` 或更低。
- d=7 新主模板采用：
  - 等价变形伪装比较。
  - 净倍率判断比较。
  - 合并同类结构比较。
- d=7 各模板必须覆盖 `>` / `<` / `=`，通过同结构偏移参照实现，不靠具体精算阈值。
- d=7 禁止需要算出具体小数结果才能比较的题。
- d=8 只扩同类陈述，不跨入 estimate / round / floor-ceil。
- d=8 单选 `对 / 错` 概念池从 13 条扩到约 24 条，对错尽量均衡。
- 每条 explanation 必须包含规则说明、反例或边界条件。

## 五、技术实现

### 5.1 拆分 compare 生成函数

建议将现有 `generateCompareExpr` 拆成两层：

- `generateCompareBasicExpr(difficulty, id)`：服务 `d=4-6`，承接旧一步表达式比较。
- `generateCompareAdvancedExpr(difficulty, id)`：服务 `d=7`，只生成二步结构 / 误导性比较。

`generateCompare(difficulty, id)` 路由调整为：

```ts
if (difficulty >= 8) return generateCompareConcept(difficulty, id);
if (difficulty >= 7) return generateCompareAdvancedExpr(difficulty, id);
if (difficulty >= 4) return generateCompareBasicExpr(difficulty, id);
return generateCompareLow(difficulty, id);
```

### 5.2 d=7 主模板

#### 等价变形伪装比较

核心：先识别等价变形，再与略大 / 略小的同结构表达式比较。

示例：

- `a ÷ 0.25 ○ a × 4` → `=`
- `a ÷ 0.25 ○ a × 4.1` → `<`
- `a ÷ 0.25 ○ a × 3.9` → `>`
- `a × 0.2 ○ a ÷ 4.8` → `<`，因为 `×0.2 = ÷5`，而 `÷4.8` 更大

禁止：生成需要算出 `a × 4` 具体值才能判断的题。

#### 净倍率判断比较

核心：先把连续乘除化成整体倍率，再比较。

示例：

- `a × 0.8 ÷ 0.4 ○ a` → `>`，整体 `×2`
- `a ÷ 1.25 × 2.5 ○ a` → `>`，整体 `×2`
- `a × 1.2 ÷ 0.6 ○ a × 2` → `=`

#### 合并同类结构比较

核心：先合并同类表达式，再用倍率与 1 或同结构参照比较。

示例：

- `a × 0.6 + a × 0.4 ○ a` → `=`
- `a × 0.6 + a × 0.5 ○ a` → `>`
- `a × 0.6 + a × 0.3 ○ a` → `<`
- `a × 0.6 + a × 0.5 ○ a × 1.2` → `<`

禁止：`37.5 × 0.6 + 37.5 × 0.25 ○ 30` 这类必须算出具体小数结果才能比较的题。

### 5.3 d=8 单选概念池扩充

当前单选池约 13 条。目标扩到约 24 条。

扩大方式：

- 优先复用现有多选概念池中的陈述。
- 每条转换为 `{ stmt, truth, explain }`。
- 对错尽量均衡，目标约 12 / 12。
- 按五类组织：
  - 乘法大小变化。
  - 除法大小变化。
  - `0` / `1` 边界。
  - “一定 / 所有 / 任何 / 都”绝对化陷阱。
  - 条件缺失陷阱。

explanation 规则：

- 错题必须给反例或指出缺失条件。
- 对题必须说明成立条件，避免学生只背结论。
- 对“任何 / 所有 / 一定”题，解释必须点出量词为何成立或为何被反例推翻。

### 5.4 诊断脚本调整

`scripts/diagnose-phase3-question-quality.mjs` 不再把 A02 compare 的二选一形态视为问题。

改为输出：

- d=7 compare 样本中三类模板覆盖情况。
- d=7 `>` / `<` / `=` 答案分布。
- d=8 单选概念池规模。
- d=8 对错比例。
- d=8 explanation 完整性检查：
  - 是否包含反例、条件、规则说明中的至少一种。
- d=8 完全重复样本统计。

## 六、TDD 任务拆解

### Task 1：d=7 不再生成一步 compare

测试点：

- 抽样 d=7 `compare` 题。
- prompt 不应匹配旧一步结构主力：`a × b ○ a × c`、`a ÷ b ○ a ÷ c`、`a × b ○ a`。
- 每道题带有可识别的 advanced pattern 标记或可通过 helper 分类。

建议在题目 `data` 中增加内部调试字段需谨慎；若不加字段，则测试使用 prompt 分类 helper。

### Task 2：d=7 覆盖三类主模板与三种答案

抽样 d=7：

- 出现等价变形伪装、净倍率判断、合并同类结构三类。
- 答案集合包含 `>` / `<` / `=`。
- explanation 包含“等价”“整体倍率”“合并”之一。

### Task 3：d=8 单选池扩到约 24 条

测试点：

- 单选池规模至少 24。
- `对` / `错` 数量差不超过 4。
- 每条 explanation 非空，且包含规则、反例或边界条件。
- d=8 仍返回 `options: ['对', '错']` 的判断题。

### Task 4：诊断脚本更新

运行：

```powershell
node scripts\diagnose-phase3-question-quality.mjs
```

输出不再将 A02 compare 二选一列为 option issue，而是列入 compare quality summary。

## 八、风险与回滚

- d=7 prompt 可能变长，需控制表达式数量和小数位。
- 若 d=7 模板分类测试过脆，优先抽取本地 helper，而不是在测试中写复杂正则。

- 回滚时恢复旧 `generateCompareExpr` 路由，但保留 d=8 explanation 质量测试作为后续目标。
