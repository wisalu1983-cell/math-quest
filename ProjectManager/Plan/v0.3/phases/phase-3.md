## 预研收口结论（2026-04-24）

### 1. 首次登录与本地进度保护（P0）

首次登录合并不是单纯弹窗，而是 Phase 3 的最高优先级数据保护链路。

**有效本地进度判定**：

仅有 nickname / avatar 不算有效进度。满足以下任一条件才算本地有进度：

- `totalQuestionsAttempted > 0`
- 任一 topic 有闯关完成记录
- 任一 topic 有进阶进度 / 星级进度
- 有历史答题记录
- 有错题记录
- 有段位进度或活跃段位赛记录

**本地 / 云端处理规则**：

| 场景 | 行为 |
|---|---|
| 本地无有效进度，云端有数据 | 自动拉取云端 |
| 本地有有效进度，云端无数据 | 自动上传本地 |
| 本地有有效进度，云端也有数据，且属于同一账号 / 未绑定账号 | 弹合并引导 |
| 本地属于另一个账号 | 不自动合并；见"账号切换与隔离" |

合并引导只提供两个确定性选择：

- 主按钮：合并本地和云端进度
- 次按钮：使用云端进度（当前设备本地进度会被云端进度替换）

不提供"稍后再说"。登录后同步系统需要一个确定的数据归属状态。

**技术约束**：

- 合并判定完成前，不能让 `SyncEngine.initialize()` 自动 `fullSync()` 抢跑
- 完成合并 / 拉取 / 上传后，再启动 SyncEngine
- 合并完成后写入当前 Supabase user id，后续同账号刷新不再触发首次登录引导

### 2. 账号切换与隔离

不同账号数据严格独立，不能相互干扰。

同一设备从账号 A 切换到账号 B：

1. 切换前先确认账号 A 是否已同步完成
2. 若 A 有未同步数据，提示"当前账号还有未同步进度，请联网同步完成后再切换账号"
3. 若 A 已同步，允许登出 / 切换
4. 登录 B 后，只拉取 B 的云端数据
5. B 云端无数据时，按 B 的空账号处理，不继承 A 数据
6. 不提供把账号 A 本地数据合并进账号 B 的入口

### 3. 同步状态展示

同步状态只在用户已登录后展示。

| 状态 | 展示规则 |
|---|---|
| 未登录 | 不显示 |
| 首次登录合并完成前 | 不显示"已同步"，由合并流程表达状态 |
| syncing | 仅真实同步动作进行中显示，例如首次同步、合并后写入、答题后推送、网络恢复补同步、远端变更拉取 |
| synced | 同步完成后显示 |
| offline | 已登录但离线时显示"离线，进度会保存在本地" |
| error | 显示"同步出错，稍后自动重试" |

展示位置：

- Home 顶部：轻量图标 / 短状态
- Profile 账号区域：完整状态文字
- Practice 不显示同步状态，避免干扰答题

Phase 3 不做手动重试按钮。同步失败走指数退避自动重试；用户不承担同步控制责任。登录页 Magic Link 发送失败仍允许重新发送。

### 4. Supabase 未配置 / 服务不可用

开发阶段必须配置 `.env` 后测试账号同步。

线上阶段：

- 登录页遇到 Supabase/Auth 不可用：显示"账号服务暂时不可用，请稍后再试。你仍可以先用访客模式练习。"
- 已登录后同步失败或 Supabase 不可用：显示"云同步暂时不可用，进度已保存在本地，恢复后会自动同步。"
- 闯关 / 进阶继续可用
- 段位赛启动必须能确认联网和远端活跃状态；无法确认时拒绝启动

### 5. 段位赛联网规则

| 场景 | 行为 |
|---|---|
| 离线开始新段位赛系列 | 拒绝启动 |
| 离线开始系列内下一局 | 拒绝启动 |
| 局中断网 | 当前局继续，结果先存本地，联网后同步 |
| 远端已有活跃段位赛 | 当前设备不能再开新系列，提示"请先在另一台设备完成段位赛" |
| 闯关 / 进阶 | 继续离线可用 |
