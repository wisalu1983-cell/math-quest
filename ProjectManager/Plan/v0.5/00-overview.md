## 版本定性

v0.5 的主题是 **A03 竖式体验与输入系统**。

本版本以 A03 竖式训练体验和 Practice 输入体验为主线，补齐竖式题样本质量诊断、计算输入内置键盘、过程错因反馈和竖式除法 UI 化答题能力。

## 目标

- 让竖式题样本更符合“必须通过竖式训练完成”的学习目标，排除普通五年级学生大概率能心算的样例。
- 为整数除法、小数除法、取近似、循环小数建立可交互、可判定、可扩展的长除法答题 UI。
- 建立全量计算字符输入的内置键盘，让移动端默认不弹系统键盘；系统键盘呼出能力保留在工程层，但首版 UI 屏蔽。
- 修复多行乘法竖式过程格 / 训练格错误时统一失败面板缺少错因的问题，为除法竖式严格判定打基础。
- 将 `BL-012`（特定账号数据查询后台页面）拆分到 v0.7，避免账号 / Supabase 后台风险拖大 v0.5 / v0.6 的学生端体验与 A03 优化范围。

## 范围边界

纳入 v0.5：

- `BL-009` 闯关竖式题目需排除纯口算样例。
- `BL-011` 计算输入内置键盘与移动端默认键盘策略。
- `BL-010` 竖式除法 UI 化答题功能。
- `ISSUE-067` 多行乘法竖式判错面板缺少过程 / 训练格错因。
- `ISSUE-069` reverse-round 填空题答案口径冲突。
- `BL-017` 题型生成器硬编码样例池审计判断（只做审计与影响判断，不承诺全量修复）。

不在 v0.5 当前预研结论中直接承诺：

- 重做整个错题本；本版本只要求记录错题和错因，作为未来分析数据来源。
- 让历史记录页面展示错因；历史记录底层可保留字段，但 UI 不展示。
- 依赖 `virtualKeyboardPolicy` 作为跨浏览器核心方案；Safari 稳定路径必须绕开该依赖。
- `BL-012` 特定账号数据查询后台页面；该需求拆分至 v0.7。
- 在详细设计前提前改写任何 `current.md`。
