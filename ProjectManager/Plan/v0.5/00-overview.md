## 启动依据

v0.5 基于以下事实源启动：

- v0.4 已发布，A03 竖式计算 current spec 已建立，乘法竖式 UI 与统一结果 UI 已成为后续竖式能力的直接基础。
- 2026-04-27 ~ 2026-04-28 用户提出一批新需求，并通过对话预研完成关键产品决策。
- `Backlog.md` 已记录 `BL-009` ~ `BL-012`；其中 `BL-009`、`BL-011`、`BL-010` 纳入 v0.5，`BL-012` 拆分至 v0.6。
- `ISSUE_LIST.md` 已记录 `ISSUE-067`，并纳入 v0.5。
- 本版本启动目标是进入 Phase 1 开工对齐前的稳定状态：版本包、范围、阶段、验收口径、风险边界先行。

## 版本定性

v0.5 的主题是 **A03 竖式体验与输入系统**。

本版本以 A03 竖式训练体验和 Practice 输入体验为主线，补齐竖式题样本质量诊断、计算输入内置键盘、过程错因反馈和竖式除法 UI 化答题能力。

## 目标

- 让竖式题样本更符合“必须通过竖式训练完成”的学习目标，排除普通五年级学生大概率能心算的样例。
- 为整数除法、小数除法、取近似、循环小数建立可交互、可判定、可扩展的长除法答题 UI。
- 建立全量计算字符输入的内置键盘，让移动端默认不弹系统键盘，并保留可靠的系统键盘兜底入口。
- 修复多行乘法竖式过程格 / 训练格错误时统一失败面板缺少错因的问题，为除法竖式严格判定打基础。
- 将 `BL-012`（特定账号数据查询后台页面）拆分到 v0.6，避免账号 / Supabase 后台风险拖大 v0.5 的学生端体验 release gate。

## 范围边界

纳入 v0.5：

- `BL-009` 闯关竖式题目需排除纯口算样例。
- `BL-011` 计算输入内置键盘与移动端默认键盘策略。
- `BL-010` 竖式除法 UI 化答题功能。
- `ISSUE-067` 多行乘法竖式判错面板缺少过程 / 训练格错因。

不在 v0.5 当前预研结论中直接承诺：

- 重做整个错题本；本版本只要求记录错题和错因，作为未来分析数据来源。
- 让历史记录页面展示错因；历史记录底层可保留字段，但 UI 不展示。
- 依赖 `virtualKeyboardPolicy` 作为跨浏览器核心方案；Safari 稳定路径必须绕开该依赖。
- `BL-012` 特定账号数据查询后台页面；该需求拆分至 v0.6。
- 在详细设计前提前改写任何 `current.md`。
