## 2. 最近确认的产品口径

- 除法必须使用专门的长除法竖式 UI，不能退回通用训练格。
- 每个数字一格，数位上下对齐；小数点是格间符号，不占数字格。
- 长除号左侧采用接近真实试卷的右括号式弧线，不用笔直竖线。
- 竖式板格子随答题进度逐步出现，不一开始展示所有格子。
- 小数 ÷ 小数分成扩倍环节和实际计算环节；扩倍环节先在同一个转换区填写：
  - 除数扩大几倍
  - 转换后除数
  - 转换后被除数
- 用户点击“确认扩倍”后才提交扩倍环节；扩倍确认通过后，竖式板根据转换后除数 / 被除数动态生成。
- 扩倍环节三个输入格的小数点键始终可用，`100.0` / `24.0` / `1560.0` 这类等值小数按数值归一处理。
- 转换后除数必须可归一为整数，转换后被除数兼容整数或仍是小数。
- 商小数点规则已简化：小数 ÷ 整数由系统预置并和被除数对齐；整数 ÷ 整数只在个位处理完仍需继续小数时自动出现；小数 ÷ 小数通过转换后除法复用这两种模式。
- 非高档：扩倍提交有误时，只提示字段填写有误，不给正确值，并停留在转换区等待再次确认。
- 高档：扩倍提交有误时，本题直接进入失败反馈，并展示具体错因；不再做转换区错误重试循环。
- 提交时仍走通用失败反馈，所有触发错因统一展示。
- 循环小数：循环节最多 3 位；当前预览用 `14 ÷ 135` 验证严格展示串 8 字符上限下的极限显示，非循环部分填写完整前缀 `0.1`，循环节填写 `037`，标准格式答数按中国常见循环点格式显示，竖式板部分至少让循环节出现 2 次后才算完成。

## 3. 当前预览实现重点

- `LongDivisionUiReviewPreview.tsx` 中 `buildBoardModelFromInput()` 会基于用户填写的转换后被除数 / 整数除数生成长除法轮次。
- 小数 ÷ 小数的 setup 字段已从竖式头部移到同一个 `data-conversion-panel="true"` 转换区，并用 `conversionSubmitted` / `conversionConfirmed` 区分“扩倍环节”和“实际计算环节”。
- 竖式头部不再让学生重复填转换值，而是展示转换区输入派生出的除数和被除数。
- 扩倍环节主按钮为“确认扩倍”；通过后主按钮切换为“提交答案”。
- 若转换后除数或转换后被除数填错，非高档预览稿会在转换区提示“转换后除数填写有误”或“转换后被除数填写有误”，不直接告诉正确值，也不进入竖式板；高档则直接展示失败反馈明细。
- 若转换后除数仍为小数，例如继续填 `0.24`，预览稿会在转换区提示“转换后除数必须是整数”，并不进入竖式板。
- `1560 ÷ 24` 这类转换后被除数为整数且除尽的竖式，商行不再提前显示小数点。
- 小数 ÷ 小数的“转换后整数”和“转换后仍有小数”已合并到同一个预览场景，用场景内切换按钮查看。
- 扩倍环节三个输入格已统一允许输入小数点，并按数值归一比较；小数点可用性不再绑定标准答案是否含小数。
- 中档即时提示只在扩倍提交后展示；高档隐藏即时提示并直接失败。
- 同一训练 / 结构化区内的输入框已统一宽度，不随标准答案位数变宽变窄。
- 失败反馈面板已改为根据当前实际 wrong fields 生成，不再只展示静态示例；主说明按“过程错 / 扩倍字段错 / 结果字段错 / 混合错”四类输出短句，只说明未通过和错误位置，不出现行动要求。
- 竖式板已加入通用自适应压缩档位：字号最低 `12px`、间距压缩、除号 / 除数区压缩、板 / 页面边界压缩依次生效；当前极限循环小数样例在 390px CSS 宽度下不需要左右滑动。
- 循环小数结果格已新增“标准格式答数”栏，根据用户输入实时生成中国循环点格式；页面不再显示 `0.1(037)` 这类括号格式。
