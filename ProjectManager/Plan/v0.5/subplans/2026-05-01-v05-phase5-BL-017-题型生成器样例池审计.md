### 审计摘要

采样方式：对 A01~A08 相关 generator 的各 subtype 按代表难度生成 240 题，统计 `type + prompt + solution.answer` 的唯一数和 top duplicate。过程脚本位于忽略目录 `.agent-tmp/v05-phase5/generator-repeat-audit.ts`，关键结果已摘录如下。

| 风险项 | 证据 | 判断 |
|---|---|---|
| `vertical-calc` 高档 `cyclic-div` | 修复前 240 题仅 3 个唯一题干，top duplicate 89；修复后约 204 个唯一题干，top duplicate 4 | v0.5 新增发布面，高风险，已小范围修复 |
| `operation-laws` 多个概念 subtype | 多数 subtype 唯一数 3~16，重复率 0.93+ | 系统性概念题模板池问题，转 v0.6，不阻塞 v0.5 |
| `number-sense/floor-ceil` | d=3 唯一数 8，d=7/9 唯一数 20 | 旧 A02 情境池有限，转 v0.6 生成器样本质量优化 |
| `decimal-ops` 部分低 / 中档 subtype | `add-sub` d=3 唯一数 17，`trap` d=7 唯一数 62 | 旧题型重复风险，转 v0.6 |
| `mental-arithmetic/order` 高档 | d=9 唯一数 21，top duplicate 24 | 旧技巧池有限，转 v0.6 |
| `multi-step` 部分 subtype | d=7 多项唯一数约 125~137，但 top duplicate 15~21 | 中风险，转 v0.6 统一治理 |

### 关键决策与备选方案

| 选项 | 结论 | 原因 / 后果 |
|---|---|---|
| 选定方案：v0.5 只修 `vertical-calc/cyclic-div`，其余系统性问题转 v0.6 | 采用 | 当前新增发布面风险最高且修复面小；旧题型问题分散，Release Gate 内全修风险高。 |
| 未采用方案：v0.5 全量扩容所有高重复池 | 放弃 | 会触碰多个题型的学习设计、难度梯度和 QA 面，超出 Release Gate 稳定化范围。 |
| 未采用方案：只记录审计，不修 `cyclic-div` | 放弃 | `cyclic-div` 是 v0.5 Phase 4 新增能力，3 个固定样例会直接影响本版本验收信心。 |

### 游戏 / 学习体验影响

| 项 | 结论 |
|---|---|
| 是否影响玩法 / 学习体验 | 是 |
| 玩家 / 学生目标 | 在高档循环小数长除法中获得足够多的练习变化，而不是反复遇到同 3 题。 |
| 学习目标或训练价值 | 训练循环节识别和长除法过程，而不是记住固定题。 |
| 误解 / 挫败 / 难度风险 | 随机生成需限制循环节长度，避免手机 UI 容量和认知负担失控。 |
| 体验验证方式 | 生成样本唯一数测试、长除法既有测试、build、Release Gate QA。 |

## 二、目标与范围

- 完成全题型样例池 / 重复风险审计。

- 不承诺所有题型在 240 题采样中达到统一重复率阈值。
