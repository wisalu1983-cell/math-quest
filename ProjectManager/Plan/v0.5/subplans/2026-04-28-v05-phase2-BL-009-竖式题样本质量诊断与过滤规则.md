## 一、背景

`BL-009` 的目标是让 A03 闯关竖式题更符合“必须通过竖式训练完成”的学习目标，排除普通五年级学生大概率可以心算完成的样例。用户已明确纠偏：`888 ÷ 4`、`844 ÷ 4`、`208 ÷ 4`、`672 ÷ 4`、`2位数 × 1位数` 等题不应因为有轻微位值 / 商位价值就继续作为竖式训练样本保留。

Phase 2 的执行顺序是先抽样诊断，再基于证据确认强制过滤规则和落点。本计划先建立诊断脚本与口径，不在诊断前写死最终过滤阈值。

## 二、目标与范围

### 目标

- 跑固定 seed 抽样，覆盖 A03 闯关 lane、整数乘法、整数除法、不同 difficulty 档。
- 将“心算候选题”拆成机器可初筛的候选类别和需要人工复核的边界类别。
- 基于抽样结果提出第一版过滤规则、反例保护和代码落点。
- 在用户确认过滤力度后，再进入实现与自动化测试。

### 非目标

- 本轮不实现竖式除法 UI。
- 本轮不改 Practice 输入、错因反馈或错题本。
- 不在诊断前直接修改生成器规则。
- 不提前回写 `current.md`。

## 三、诊断口径

### 抽样命令

```bash
node scripts/diagnose-bl009-vertical-samples.mjs
```

### 抽样范围

| 范围 | 样本量 | 覆盖目的 |
|---|---:|---|
| A03 campaign 每个 level | 每关 80 个 session × 该关题量 | 观察真实闯关 lane 下心算候选比例。 |
| A03 focused generator bucket | 每个重点 subtype / difficulty 500 题 | 定位问题来自具体 subtype 还是 campaign lane 混合。 |

### 初筛类别

| 类别 | 当前处理 | 说明 |
|---|---|---|
| `mul-2d-by-1d` | 强候选 | 用户已点名 `2位数 × 1位数` 不应作为竖式训练样本。 |
| 除一位整数 D0：逐段整除型 | 强候选 | `888 ÷ 4`、`844 ÷ 4`、`208 ÷ 4` 这类中间无余数传递，接近拆数口算，应过滤。 |
| 除一位整数 D1：单次余数传递，最后整除 | 低优先级保留 | 可作为低档过渡题，但不作为主力样本。 |
| 除一位整数 D2：多次余数传递，最后整除 | 主力保留 | `672 ÷ 4`、`668 ÷ 4` 这类需要维护多次余数、落位、再除，符合竖式训练目标。 |
| 除一位整数 D3：商中间 0 | 主力保留 / 补强 | 商位占位是竖式除法关键训练点，应作为低档后段主力样本之一。 |
| `mul-3d-by-1d` | 待复核 | 可能仍有竖式价值，但需检查是否存在大量低负担样例。 |
| `div-4d-by-1d-integer` | 待复核 | 当前低档后段使用四位数除以一位数；是否保留取决于样本负担和用户判断。 |
| `add-low-load-no-carry` / `sub-low-load-no-borrow` | 待复核 | 加减法若完全无进退位，可能低于竖式训练目标，但本轮优先级低于乘除。 |

### 已确认过滤规则

| 优先级 | 规则 | 处理 |
|---|---|---|
| P0 | 排除 `mul-2d-by-1d` | 进入第一批生成器过滤。 |
| P0 | 排除除一位整数 D0 逐段整除型 | 进入第一批生成器过滤；覆盖 `888 ÷ 4`、`844 ÷ 4`、`208 ÷ 4` 这类接近口算的样例。 |
| P0 | 低档除法以 D2 / D3 为主 | D2 多次余数传递、D3 商中间 0 是低档除法主力样本；D1 单次余数传递只作少量过渡。 |
| P1 | 复核 `mul-3d-by-1d` | 先不直接过滤全部，需抽几组人工判断是否按“含连续进位 / 位数负担”细分。 |
| P1 | 复核 `div-4d-by-1d-integer` | 先不直接过滤全部，需确认四位数除以一位数是否仍可心算完成，或只过滤明显整百 / 整十 / 易分解样例。 |
| P2 | 复核无进退位低档加减 | 暂不进入第一批实现；等乘除规则稳定后再决定是否约束“低档加减必须有进退位”。 |

### 初步落点判断

- 第一批 P0 规则应落在 `src/engine/generators/vertical-calc.ts`，因为 focused generator 证明问题来自低档 `int-mul` / `int-div` 生成器，而不是单个 campaign level 的局部配置。
- `vertical-calc-S1-LB-L1` 当前 100% 命中旧强候选口径；实现时需把除法从 `三位数 ÷ 一位数整数商` 进一步拆成 D0-D3：过滤 D0，主动生成 D2 / D3 为主的低档除法样本。
- d4 / d5 乘除是过滤力度决策点：若用户希望严格排除所有可心算题，P1 可能要继续细分；若首版保守推进，可以先只落 P0。
