## 1. 审核目标

本稿只确认 `BL-010` 的 UI 方案，不提前实现正式长除法判定逻辑。

审核范围：

- 长除法核心板形态：采用专用竖式 UI，商位在长除号上方，除数 / 被除数在竖式主体内，乘积、减线、余数、落位 / 补 0 都在同一块竖式板里完成。
- 数位对齐：题干被除数、商位、乘积、减后工作数全部使用同一套固定数字列；每格只承载一位数字。
- 长除号参考：按标准试卷 / worksheet 的长除号处理，除数在左，左侧采用右括号式弧线分隔，细横线覆盖被除数上方，避免直竖线或厚重圆角“厂字头”。
- 五类题型 UI：整数除法、小数 ÷ 整数、小数 ÷ 小数、取近似、循环小数。
- 输入交互：slot 顺序、自动换格、桌面 Tab、内置键盘。
- 反馈交互：过程格类别反馈、训练 / 结构化字段明细反馈。
- 移动端容量：375px 竖屏下，答题区、反馈面板、固定底部键盘不互相遮挡。

## 2. 推荐方案

采用“专用长除法竖式板 + 外围训练格 / 结果格”的方案。

- 整数除法和转换后的整数除法，核心答题都进入专用 `LongDivisionBoard`，不把商、乘、减、落拆成普通训练格。
- 商位输入在长除号上方；乘积、相减结果、落位 / 补 0 后新工作数在竖式主体里输入。
- 竖式主体内只保留数字格、减号、横线和小数点；“第几轮”“商乘减比落”等教学提示放到旁侧状态或无障碍说明，不压进笔算表内。
- 竖式板不一次性展示全部待填格；只显示当前已到达 / 已填写的字段，后续乘积行、减线、余数与落位行、下一轮商位按输入顺序逐步向下长出。
- 竖式板所有数字格统一略窄，小数点作为独立符号放在相邻数字格间隙中，保证左右都有清晰留白。
- 数字列间隙分两套：普通相邻数字列使用常规间隙；存在小数点时，只有小数点左右两个数字格之间使用小数点专用间隙，并把小数点放进该间隙轨道，保证桌面和移动端缩放后仍与数字格中心线对齐。
- 长除号弧线使用独立括号列；竖式板上的除数 / 被除数由转换区输入派生，被除数首位从括号列右侧开始，不能与弧线重叠。
- 计算前训练格和竖式板都按主答题区中心线居中，避免板面偏左、右侧空白过大。
- 非终止轮不再出现“余数格 + 新工作数格”的重复填写；减线下方同一行直接填写余数与落位后的工作数，例如 `9 - 8` 后直接在下一行填 `13`，再进入下一轮。
- 小数 ÷ 整数中，商的小数点必须与被除数小数点垂直对齐，并作为独立符号放在两个答案数字格之间，不占用数字格。
- 小数 ÷ 小数中，前置转换区先填写除数扩大的倍数，再填写转换后的除数 / 被除数；竖式板根据用户输入的转换后除数 / 被除数自动生成。
- 中档只在答题过程中对转换值错填给短提示，高档过程不提示。
- 提交答案时仍走通用失败反馈：扩倍倍数、转换后的除数 / 被除数、长除法过程格等所有触发的错因统一展示。
- 扩倍后的被除数可以是整数，也可以仍是小数；转换区允许填写小数点，竖式板按用户输入固定小数点位置并生成数字列。
- 历史轮次留在同一块竖式板内；当前 active slot 所在轮次高亮。
- 前置训练格只用于小数 ÷ 小数扩倍；后置结构化结果格只用于取近似、循环小数等核心计算后的结果表达。
- 内置键盘复用 Phase 3 `MathInputSlot` 协议，题型组件只声明 slot。

选择原因：

- 375px 手机竖屏仍以专用竖式板为主，优先通过整体缩放让算式在内置键盘上方完整可见；缩放下限按竖式板基础字号 `16px`、最小可读字号 `14px` 计算，即最小 scale `0.875`，低于该值不再继续缩小。
- 符合 Phase 4 已确认的全过程填写：学生仍要填商位、乘积、余数、落位 / 补 0。
- “比”保留为系统校验节点，不增加无意义输入。
- 与 Phase 3 已实现的固定底部键盘、自动换格、结构化错因能直接衔接。

## 3. 预览场景

| 场景 | 样例 | 审核重点 |
|---|---|---|
| 整数多轮 | `936 ÷ 4` | 多次余数传递；完整“商、乘、减、比、落”顺序 |
| 商中间 0 | `824 ÷ 4` | 不够除时商 0；落下一位后形成新工作数 |
| 小数 ÷ 整数 | `5.76 ÷ 3` | 商的小数点系统预置，不作为输入格 |
| 小数 ÷ 小数 | `15.6 ÷ 0.24` | 先填扩大倍数，再在同一转换区填写 `24` 和 `1560`，竖式板据此生成 |
| 小数 ÷ 小数 · 扩倍后仍有小数 | `1.234 ÷ 0.04` | 除数转整数后，在转换区填写 `4` 和 `123.4`，竖式板据此生成小数点列 |
| 取近似 | `8.5 ÷ 3` 保留两位小数 | 算到保留位后一位，最后只填近似结果 |
| 循环小数 | `1 ÷ 6` | 整数位商 0 也由学生填写；循环节在竖式板中至少出现 2 次后，核心计算后只填非循环部分和循环节 |

## 4. 交互设计

### 4.1 输入顺序

统一顺序：

```text
小数 ÷ 小数前置扩倍训练格（仅该类题有）
→ 转换后的竖式除数 / 被除数
→ 长除法竖式板第 1 轮 商位
→ 第 1 轮 乘积
→ 第 1 轮 余数与落位后的工作数（同一行）
→ 系统校验“比”
→ 下一轮 ...
→ 取近似 / 循环小数后置结构化结果格（仅该类题有）
```

规则：

- 单位商位达到 1 位后自动进入乘积。
- 乘积、余数与落位同一行达到标准答案长度后自动进入下一 slot。
- 竖式板每次只露出当前步骤需要的格子；已完成的历史步骤保留在板内，未到达的后续轮次不提前显示。
- `delete` 不自动换格。
- 桌面 `Tab` / `Shift+Tab` 使用同一份 slot 顺序。
- 移动端默认固定底部内置键盘；不依赖系统键盘作为首选输入方式。
- 题底操作区保留统一规格的“提交”和“重置”按钮；重置清空当前题所有输入、关闭反馈，并回到第一格未填写状态。
- 进入失败反馈态后收起内置键盘，反馈面板自动滚入可见区域。

### 4.2 题型扩展格

| 题型 | 前置 / 后置格 | 展示方式 |
|---|---|---|
| 小数 ÷ 整数 | 无额外输入；商小数点系统预置 | 核心板上方给出绿色提示 |
| 小数 ÷ 小数 | 扩倍倍数；转换后的除数 / 被除数 | 同一转换区连续填写；竖式板按用户输入生成；中档错填转换值时当场给短提示，高档过程不提示；提交后错因面板仍统一展示 |
| 取近似 | 保留位数结果 | 核心板后的结果格区 |
| 循环小数 | 非循环部分、循环节 | 整数位 0 不预填，先在核心板完成 `0`、乘积 `0`、补 0 后工作数；循环节至少出现 2 次后，核心板后的结果格只表达最终循环结构 |

### 4.3 失败反馈

过程格错误：

- 展示类别，不展示正确中间值。
- 例：`商位判断错误`、`乘积填写错误`、`落位后新工作数错误`。

训练 / 结构化字段错误：

- 展示字段名、用户值、正确值。
- 例：`扩倍倍数错误：你填 10，正确是 100`。
- 小数 ÷ 小数的中档过程提示只作为答题中即时纠偏，不替代提交后的结构化错因；高档不显示过程提示，但提交后仍展示触发的结构化错因。

错题本：

- 复用同一份结构化错因摘要。

历史记录 UI：

- 不展示错因，只保留底层数据。

## 5. 移动端容量策略

- 页面不能出现全页横向滚动；专用竖式板优先整体缩放，达到最小 scale 后仍无法容纳时才允许板内受控横向滚动。
- 当前 active slot 所在轮次高亮，必要时实现自动滚动到可见区域。
- 固定底部键盘保留约 `14rem` 底部空间，避免提交按钮被遮挡；反馈态隐藏键盘，避免反馈面板与键盘抢占空间。
- 手机竖屏中，竖式板按可用宽度和键盘上方剩余高度共同计算 scale；桌面优先展示更接近纸面笔算的完整板面。
- 循环小数结果格展示长度沿子计划限制：循环节最长 3 位、非循环部分最多 2 位小数、总展示字符串不超过 8 字符。
- 循环小数竖式板的结束条件改为“循环节至少出现 2 次”。这与结果格的“循环节最长 3 位”不冲突；但它比“首次重复余数即结束”多展示一轮循环，需要在移动端容量和题池过滤中单独校验。

## 6. 已记录问题

- `BUG-PREVIEW-001`：桌面物理键盘输入后，active slot 视觉状态会自动进入下一格，但浏览器真实焦点仍留在旧 input，导致继续打字写不进下一格。预览稿已修正为桌面端 `activeSlotId` 变化时同步 focus；正式 `BL-010` 实现必须继承这条焦点规则。
