## 依赖关系

| 依赖 | 性质 | 说明 |
|---|---|---|
| `BL-011` → `BL-010` | 硬依赖 | 竖式除法 UI 会大量使用格子输入，必须复用内置键盘的输入模型。 |
| `ISSUE-067` → `BL-010` | 硬依赖 | 除法竖式采用严格过程判定，必须先具备可解释的过程 / 训练格错因反馈链路。 |
| `BL-009` → `BL-010` | 软依赖 | 样本质量诊断可独立推进，但除法 UI 上线前应避免明显口算样例进入竖式训练。 |
| `BL-017` → Release Gate | 审计门 | 样例池重复风险可能影响发布信心；v0.5 只要求完成审计判断和分流，不承诺全量修复。 |
| Release Gate → v0.5 全部子项 | 硬依赖 | v0.5 收口前必须完成自动化、移动端浏览器、QA 与 PM 一致性检查。 |
| `BL-012` 与 v0.5 / v0.6 当前范围 | 拆分关系 | `BL-012` 属于账号 / Supabase / 后台查询维度，进入 v0.7，不参与 v0.5 release gate 或 v0.6 A03 优化范围。 |

## 当前边界

### 纳入

- A03 竖式题样本诊断与过滤规则。
- 动态长除法答题 UI。
- 小数 ÷ 整数、小数 ÷ 小数、取近似、循环小数的训练格设计。
- 全量计算字符输入的内置键盘。
- Chrome Android / Safari iOS 的移动端默认内置键盘验收。
- 多行乘法竖式和未来除法竖式的错误原因展示与错题数据记录。
- `ISSUE-069` reverse-round 模板 4 答案口径修复。
- `BL-017` 全题型硬编码样例池 / 重复风险审计判断。

### 不纳入

- 重做错题本学习闭环；本版本只记录错题与错因，不做重练流程。
- 历史记录 UI 展示错因。
- 依赖实验性浏览器 API 作为输入核心能力。
- 大范围视觉重设计；所有 UI 继承阳光版 v5 与现有 Practice 信息架构。
- `BL-012` 特定账号数据查询后台页面；该需求拆分至 v0.7。
- 后台写入、修改或修复线上数据的能力。
